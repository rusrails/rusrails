h4. Понимание +yield+

В контексте макета, +yield+ определяет раздел, где должно быть вставлено содержимое из вьюхи. Самый простой способ его использования это иметь один +yield+ там, куда вставится все содержимое вьюхи, которая в настоящий момент рендериться:

<erb>
<html>
  <head>
  </head>
  <body>
  <%= yield %>
  </body>
</html>
</erb>

Также можете создать макет с несколькими областями yield:

<erb>
<html>
  <head>
  <%= yield :head %>
  </head>
  <body>
  <%= yield %>
  </body>
</html>
</erb>

Основное тело вьюхи всегда рендериться в неименнованный +yield+. Чтобы рендерить содержимое в именнованный +yield+, используйте метод +content_for+.

h4. Использование +content_for+

Метод +content_for+ позволяет вставить содержимое в блок +yield+ в вашем макете. +content_for+ можно использовать только для вставки содержимого в именнованные yields. Например, эта вьюха будет работать с макетом, который вы только что видели:

<erb>
<% content_for :head do %>
  <title>A simple page</title>
<% end %>

<p>Hello, Rails!</p>
</erb>

Результат рендеринга этой страницы в макет будет таким HTML:

<erb>
<html>
  <head>
  <title>A simple page</title>
  </head>
  <body>
  <p>Hello, Rails!</p>
  </body>
</html>
</erb>

Метод +content_for+ очень полезен, когда ваш макет содержит отдельные области, такие как боковые панели или футеры, в которые нужно вставить свои блоки содержимого. Это также полезно при вставке тэгов, загружающих специфичные для страницы файлы javascript или css в head общего макета.
